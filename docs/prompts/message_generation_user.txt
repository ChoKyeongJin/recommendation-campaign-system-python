규칙:
- Selected Channel Policy의 prompt와 constraints를 반드시 지켜라.
- LMS는 통신사 기준 byte_count가 2000 이하가 되도록 짧게 작성하라. 한글/전각은 2Byte, ASCII는 1Byte다.
- RCS SMS는 title 14자 이하, description 100자 이하, buttons 최대 1개, button name 7자 이하를 반드시 만족하라.
- RCS SMS에서 혜택 확인, 구매, 신청, 예약처럼 행동 유도가 필요한 경우 buttons에 CTA 버튼 1개를 생성하라. 필요하지 않으면 빈 배열을 사용하라.
- RCS 광고 메시지 title에는 '(광고)'를 포함하고, description 끝에는 '무료 수신거부' 문구를 포함하라.
- Campaign Context의 offer 또는 Existing Message Examples의 message_text 안에서 확인되는 혜택만 사용하라. 확인되는 혜택이 없으면 혜택 표현과 used_offer를 만들지 마라.
- source_campaign_id는 Campaign Context에 있는 캠페인 중 하나만 사용하라.
- used_offer는 선택값이다. 메시지에서 Campaign Context의 offer를 직접 근거로 사용한 경우에만 원문 그대로 입력하고, 그렇지 않으면 null로 둬라.
- 날짜나 마감 표현은 Campaign Context의 end_date가 있을 때만 구체적으로 말하라.
- Tone And Manner Rules를 적용해 세 메시지의 설득 포인트와 문장 구조를 다르게 작성하라.
